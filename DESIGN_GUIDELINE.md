# UI/UX 디자인 가이드라인: 김대필 (ContextWrite)

본 문서는 김대필(ContextWrite) 크롬 확장 프로그램의 일관된 브랜드 아이덴티티와 부드럽고 세련된 사용자 경험(UX)을 구축하기 위한 가이드라인입니다.

---

## 1. 브랜드 컬러 & 타이포그래피 (Brand Identity)

### 1.1 브랜드 컬러 팔레트
- **Primary Color (주요 색상):** `Deep Cyber Blue` (`#1E40AF`)
  - 신뢰감과 인공지능 스마트함을 나타내는 짙은 파란색입니다. 메인 로고, 모드 셀렉터, 기본 버튼에 적용합니다.
- **Secondary Color (보조 강조 색상):** `Humanize Mint` (`#0D9488`)
  - 자연화(Humanize) 기능 전용 강조 컬러입니다. 자연화 시작 버튼 및 변환된 어휘 하이라이팅에 사용됩니다.
- **Neutral Colors (무채색 계열):**
  - Background: Pure White (`#FFFFFF`) / Light Gray (`#F9FAFB`)
  - Text: Dark Slate (`#1F2937`) / Medium Slate (`#4B5563`)
  - Borders: Muted Gray (`#E5E7EB`)

### 1.2 타이포그래피 규격
- **Font Family:** `Arial` 또는 시스템 기본 산세리프(Sans-serif) 폰트를 적용합니다.
- **Line Spacing (줄 간격):** 가독성 보장을 위해 본문 텍스트 영역의 줄 간격은 반드시 **1.25**로 고정합니다.
- **Font Size:**
  - Header Title: 16px (Bold)
  - Content Text / Editor Input: 14px
  - Small Label / Metrics: 12px

---

## 2. 레이아웃 구조 (사이드바 메인 인터페이스)
확장 프로그램의 메인 인터페이스는 브라우저 우측에서 부드럽게 미끄러져 나오는 **슬라이드인(Slide-in) 구조의 사이드바** 형태를 취합니다.

### 2.1 영역별 디자인 규칙
1. **헤더 영역 (Header Area):**
   - 크롬 확장 프로그램 로고와 우측에 자동 모드 스위칭 작동 상태를 보여주는 토글 스위치 및 현재 신뢰도(Confidence Score %) 표시.
2. **모드 셀렉터 (Mode Selector):**
   - 드롭다운 스타일의 선택창으로 현재 감지된 모드(예: 이력서/자소서 모드)를 직관적으로 변경할 수 있어야 함.
3. **에디터 영역 (Editor Area - 비교 뷰):**
   - **Side-by-Side 뷰:** Humanize 적용 시 에디터 영역이 좌우(또는 상하) 2분할 애니메이션으로 나뉩니다.
   - **하이라이팅:** `Original AI 초안`과 `Humanized 결과물`이 대조되어 보여지며, 풍부하게 변경된 어휘나 문장은 **연한 민트색 배경색**(`rgba(13, 148, 136, 0.1)`)으로 부드럽게 하이라이트 처리됩니다.
4. **분석 대시보드 (Dashboard):**
   - AI 탐지 예측 점수 및 ATS 매칭 점수를 실시간으로 표현하는 게이지 바(Gauge Bar)를 포함합니다. 60% 이하 등 안전 영역일 시 녹색 계열, 위험 시 적색 계열 피드백 제공.
5. **푸터 영역 (Footer Actions):**
   - 웹페이지에 삽입할 수 있는 "웹페이지에 삽입" 버튼 및 로컬 소장용 "DOCX 다운로드" 버튼 등을 배치합니다.

---

## 3. 웹페이지 DOM 침투 디자인 (인라인 요소)
- **플로팅 삽입 버튼 (Mini Floating Button):**
  - 사용자가 활성화한 `textarea` 또는 `contenteditable` 입력 요소의 **우측 하단**에 자그마한 ContextWrite 미니 플로팅 아이콘이 자연스럽게 나타납니다.
  - 마우스를 올렸을 때(Hover) 미세하게 크기가 커지는 마이크로 애니메이션을 제공하며, 클릭 시 사이드바를 통하지 않고도 가장 최근에 변환한 자연화 텍스트가 즉시 주입됩니다.

---

## 4. 디자인 제약 및 일관성
- **셀 패딩(Cell Padding):** 표 형태나 리스트를 표시할 때는 셀 내부 여백을 최소 12px 이상 확보하여 시각적인 답답함을 최소화합니다.
- **여백을 통한 구획 구분:** 불필요하게 가로선을 많이 넣지 말고, 배경색의 음영 차이(`background-color`)와 바깥 여백(`margin`)을 활용하여 깔끔하고 미니멀한 레이아웃을 구현합니다.
